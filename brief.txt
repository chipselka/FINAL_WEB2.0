==============================================================
SITE: LALI — Interactive Floral Atelier
==============================================================

ROLE / CONTEXT
--------------
Role: Creative Director, UX/UI Designer, and 3D Web Product Designer for a premium floral atelier.

Context:
LALI is not a basic flower shop. It is a modern floral atelier where bouquets are treated as personal compositions, not simple products. The website should feel elegant, emotional, interactive, and memorable.

The main idea of LALI is that flowers are personal. A client can choose ready-made stylish bouquets, subscribe to regular flower deliveries, or build a custom bouquet in a 3D interactive space.

Target client:
- People who want aesthetic, stylish bouquets
- Young professionals
- Gift buyers
- Romantic occasions
- People who love design and personalization
- Clients who want flowers to feel special, not generic

Brand feeling:
Premium, romantic, modern, soft, artistic, and interactive.

This website must NOT feel like a basic ecommerce flower shop.
It should feel like a digital floral atelier with a unique 3D experience.


==============================================================
BRAND IDENTITY
==============================================================

Brand name:
LALI

Tagline:
“Arrange your own story.”

Brand concept:
LALI is a modern floral atelier that lets clients express emotion through flowers. The site combines curated bouquets, floral subscriptions, and a custom 3D bouquet builder.

Brand voice:
Soft, elegant, polished, and emotional, but not overly dramatic.

Voice rules:
- Use simple, beautiful wording.
- Avoid generic ecommerce language.
- Do not make it feel like a discount store.
- Use words like:
  “composition”
  “arrangement”
  “seasonal stems”
  “crafted”
  “atelier”
  “signature bouquet”
  “custom bouquet”
  “floral ritual”

Example phrases:
- “Flowers, arranged around your feeling.”
- “Build a bouquet that feels personal.”
- “A floral ritual delivered to your door.”
- “Choose the mood. We compose the moment.”
- “Not just flowers — a composition.”


==============================================================
PURPOSE
==============================================================

The website should allow users to:

1. Browse ready-made stylish bouquets
2. Choose bouquets by occasion or mood
3. Buy a flower subscription for 3, 6, or 12 months
4. Build a custom bouquet on a separate interactive 3D page
5. Place each flower freely inside the bouquet
6. Rotate the bouquet 360 degrees
7. See the total price update live
8. Request or place an order for the final bouquet

==============================================================
PAGE 4: 3D BUILD YOUR BOUQUET
==============================================================

This is the most important page.

Purpose:
Let users create their own bouquet in a 3D interactive space.

Page title:
“Build Your Bouquet”

Subtitle:
“Place every stem exactly where you want it.”

Layout:
Three-column layout.

LEFT PANEL:
Flower Library

Include flower options such as:
- Rose
- Tulip
- Peony
- Lily
- Orchid
- Ranunculus
- Baby’s Breath
- Eucalyptus
- Hydrangea
- Anemone

Each flower item should show:
- Small image or 3D thumbnail
- Flower name
- Color options
- Price per stem
- Drag handle or “Add” button

CENTER:
3D Bouquet Canvas

Requirements:
- Use Three.js or WebGL
- Bouquet must be visible in 3D
- Client can rotate the entire bouquet 360 degrees
- Client can zoom in and out
- Client can pan slightly
- Client can drag flowers into the bouquet
- Each flower should be a separate 3D object
- User can place each flower anywhere they want in 3D space
- User can reposition flowers freely
- User can rotate individual flowers
- User can remove selected flower
- User can reset bouquet
- User can preview bouquet from front, side, and top
- No fixed snapping only; allow free placement
- Optional soft snapping can be available, but free movement is required

Controls:
- Orbit controls for rotating bouquet
- Drag controls for moving flowers
- Transform controls for selected flowers
- Buttons:
  - Rotate bouquet
  - Reset view
  - Top view
  - Front view
  - Remove selected
  - Clear all

Visual style of 3D scene:
- Cream background
- Soft studio lighting
- Subtle shadow under bouquet
- Burgundy interface accents
- Bouquet should feel realistic and elegant

RIGHT PANEL:
Bouquet Summary

Show:
- Selected flowers list
- Quantity of each flower
- Individual price
- Total price
- Estimated size
- Button: “Order Custom Bouquet”
- Button: “Save Design”

Live pricing:
- Every added flower updates the total price
- Every removed flower updates the total price
- Quantity changes update total price

Example flower prices:
- Rose: $5
- Tulip: $4
- Peony: $8
- Lily: $7
- Orchid: $10
- Ranunculus: $6
- Baby’s Breath: $3
- Eucalyptus: $4
- Hydrangea: $9
- Anemone: $6

Important:
The 3D bouquet builder is the main wow feature.
It must not be a simple form.
It must feel interactive, visual, and fun.


==============================================================
3D BUILDER TECHNICAL REQUIREMENTS
==============================================================

Use:
- React
- Three.js
- @react-three/fiber
- @react-three/drei

Important components:
- Canvas
- OrbitControls
- TransformControls
- DragControls or custom drag logic
- GLTFLoader if using 3D models
- State management for selected flowers and price total

Functionality:
- Add flower to scene
- Select flower
- Move selected flower
- Rotate selected flower
- Delete selected flower
- Rotate entire bouquet 360 degrees
- Zoom in/out
- Update price dynamically

Data structure example:
Each flower object should include:
- id
- name
- price
- color
- model
- position
- rotation
- scale

If real 3D flower models are not available:
Use elegant placeholder 3D shapes temporarily, but structure the code so models can be replaced later.

Placeholder 3D idea:
- Stem: thin green cylinder
- Flower head: small layered petal-like shapes
- Leaves: small flat shapes
- Bouquet wrap: cream cone or ribbon base

The 3D bouquet must still be interactive even with placeholders.
